## ผมมีข้อมูลสำหรับ Employy_data_WFH _&_ Leave_work ที่เป็นรูปแบบของ Excel ของผมอยู่ต้องการนำเอาข้อมูลนี้ขึ้นระบบโดยระบบจะแบ่งการเรียงคอลลัม ดังนี้
แบ่งออกเป็น 2 ส่วนคือ Data_Leave_work (สำหรับข้อมูลวันลางาน)
- รหัสพนักงาน : ใช้สำหรับการเชื่อมข้อมูลพนักงานว่าคนไหนเป็นคนขอลาเข้ามาในระบบ
- วันที่ขอลา : สำหรับเก็บข้อมูลว่าพนักงานส่งข้อมูลเข้ามาขอลาวันที่เท่าไหร่
- วันที่ลา : สำหรับเก็บข้อมูลว่าพนักงานแจ้งขอลางานวันไหน วันที่เท่าไหร่ถึงวันที่เท่าไหร่ ยกตัวอย่างคือ 2026-01-05 ถึง 2026-01-07 
- ประเภทการลา : สำหรับเก็บข้อมูลว่าครั้งนี้ที่ขอลาเข้ามาในระบบเป็นการขอลาประเภทอะไรบ้าง กำหนดให้มี ลาป่วย , ลากิจ , ลาพักร้อน , ลาไม่รับค่าจ้าง ,ลาอื่นๆ
- หมายเหตุ : สำหรับเก็บข้อมูลว่าลากิจเนื่องจากอะไร ลาอื่นๆ เนื่องจากอะไร
- สถานะการลา : สำหรับเก็บข้อมูลว่า HR , Admin ได้มีการอนุมัติการลา หรือว่าไม่อนุมัติ กำหนดให้มี อนุมัติแล้ว , ไม่อนุมัติ , รออนุมัติ 
- หมายเหตุเพิ่มเติมสำหรับผู้อนุมัติ : เป็นการตอบกลับของ HR , Admin ที่กรอกสถานะการลา ถ้าไม่อนนุมัติการลาระบบจะต้องบังคับกรอกช่องนี้

Data_WFH (สำหรับข้อมูลการ WFH ของพนักงาน)
- รหัสพนักงาน : ใช้สำหรับการเชื่อมข้อมูลพนักงานว่าคนไหนเป็นคนขอลาเข้ามาในระบบ
- วันที่ขอ WFH : ใช้สำหรับเก็บข้อมูลว่าพนักงานขอ WFH เข้ามาเมื่อไหร่
- วันที่ WFH : ใช้สำหรับเก็บข้อมูลว่าพนักงานต้องการขอ WFH วันไหน
- สถานะ WFH : สำหรับเก็บข้อมูลว่า HR , Admin ได้มีการอนุมัติการให้ WFH หรือไม่ กำหนดให้มี อนุมัติแล้ว , ไม่อนุมัติ , รออนุมัติ 
- หมายเหตุเพิ่มเติมสำหรับผู้อนุมัติ : ใช้สำหรับเก็บข้อมูลการตอบกลับของ HR , Admin ที่กรอกสถานะการ WFH ถ้าไม่อนนุมัติการลาระบบจะต้องบังคับกรอกช่องนี้
- รายงานการทำงาน : ใช้สำหรับเก็บข้อมูลว่าพนักงานคนนี้ WFH วันนี้พนักงานจะต้องรายงานเข้าระบบว่าทำงานอะไรบ้าง ให้เขียนและส่งกลับมา

** Cursor AI ช่วยออกแบบการเก็บฐานข้อมูลส่วนนี้ให้ผมหน่อยครับ เป็นระบบการอัพเดทข้อมูลส่งข้อมูลภายในของระบบ จะต้องมีการคำนวณวันที่ลางาน จากวันที่เลือกถึงวันที่กำหนดได้

## สำหรับการพัฒนาระบบของหน้า ลางาน/WFH ดังนี้
- Role : HR , Admin จะต้องมองเห็นข้อมูลของพนักงานทุกคน > Cursor AI ช่วยออกแบบระบบของการลา/WFH และออกแบบระบบ API สำหรับการเรียกข้อมูลไม่ให้เรียกมาเยอะ โดยให้เรียกข้อมูลมาเฉพาะข้อมูลที่ต้องใช้เท่านั้นถ้าต้องการเห็นมากกว่านั้นให้เรียกส่วนอื่นเพิ่มเติมทีหลังเพื่อลดภาระการทำงานของระบบหลังบ้าน
- หน้า ลางาน/WFH จะต้องแบ่งออกเป็น 2 แถบหลัก คือ 
  - ข้อมูลพนักงานลางาน
  - ข้อมูลพนักงาน WFH
  
  ## หน้าข้อมูลพนักงานลางาน ระบบจะต้องแยกข้อมูลออกเป็น 3 แถบคือ ข้อมูลการลางาน , ข้อมูลการขอลางาน , Dashboard - ลางาน
  - HR , Admin จะต้องเห็นข้อมูลของพนักงานที่ขอลาเข้ามาในระบบได้เพื่อเลือกสถานะ อนุมัติ , ไม่อนุมัติ ได้สามารถกรอกหมายเหตุได้ว่าทำไมถึงไม่ให้ลา 
  - HR , Admin ก็เป็นพนักงานเหมือนเท่ากับว่าสามารถขอลาเข้าไปในระบบได้เหมือนกัน การอนุมัติ จะเปิดให้อนุมัติตัวเองได้ด้วย
  - Cursor AI ออกแบบระบบหน้าข้อมูลการขอลางาน ให้มองเห็นว่าในอีก 3 วันข้างหน้าพนักงานคนไหนที่ได้รับการอนุมัติการลาแล้วกำลังอยู่ในช่วง 3 วันที่จะถึงบ้างเพื่อให้รู้ว่าพนักงานจะไม่อยู่ช่วงนี้จะได้วางแผนงานกับตำแหน่งอื่นได้
  - Crusor AI ออกแบบระบบ Dashboard สำหรับการดูข้อมูลสรุปการลางานต่างๆ ได้พนักงานคนนี้ลาไปแล้วกี่วัน เหลือสิทธิ์ลากี่วัน 
  
  ## หน้าข้อมูลพนักงาน WFH ระบบจะต้องแบ่งข้อมูลออกเป็น 3 แถบคือ ข้อมูลการ WFH , ข้อมูลการขอ WFH , Dashboard - WFH
  - สิทธิที่จะสามารถขอ WFH ได้พนักงานจะต้องทำงานมาแล้วอย่างน้อย 3 เดือน
  - HR , Admin จะต้องเห็นข้อมูลของพนักงานที่ขอลาเข้าระบบได้เพื่อเลือกสถานะ อนุมัติ , ไม่อนุมัติ ได้สามารถกรอกหมายเหตุได้ว่าทำไมถึงไม่ให้ WFH
  - HR , Admin ก็เป็นพนักงานเหมือนกันเท่ากับว่าสามารถขอ WFH เข้าไปในระบบได้เหมือนกัน การอนุมัติ จะเปิดให้อนุมัติตัวเองได้ด้วย
  - Cursor AI ออกแบบระบบการเลือกข้อมูลวันขอ WFH ให้มองเห็นว่าในวันนั้นที่เลือกลา มี่พนักงานที่ขอลาในวันนี้กี่คน จำกัดให้เลือกได้ 3 คนต่อวันเท่านั้น
    - เหลือ 0/3 ให้เเสดงเป็นสีเขียว
    - เหลือ 1/3 , 2/3 ให้แสดงเป็นสีเหลือง
    - เหลือ 3/3 ให้แสดงเป็นสีแดง และกดขอ WFH ไม่ได้
  ระบบจะต้องออกแบบให้พนักงานเลือกได้ภายในเดือนนั้นขอได้สูงสุดแค่ 6 วันต่อเดือนเท่านั้น ยกเว้นตำหแน่งไอทีสามารถขอลาได้สูงสุด 16 วันต่อเดือนได้
  - Cursor AI ออกแบบระบบ Dashboard สำหรับการดูข้อมูลวัน WFH ของพนักงานแต่ละคน การรายงานของพนักงานแต่ละคนได้ ดูข้อมูลการรายงานการ WFH ย้อนหลังได้ทำเป็นสรุปข้อมูลออกมา
  

  ## หน้าเลือกข้อมูลวันขอลา และ วันขอ WFH ให้ Cursor AI ออกแบบเป็นรูปแบบของหน้าต่าง ปฏิทิน สามารถดูข้อมูลต่างๆ ได้ด้านขวาของปฏิทิน 
  ## Cursor AI จะต้องออกแบบ API ของการแสดงข้อมูลตรงนี้ให้ระบบทำงานได้อย่างรวดเร็วไม่หนักหน้าเว็บ โหลดข้อมูลมาเท่าที่จำเป็นเท่านั้น และจะต้องทำ Document สำหรับเก็บข้อมูลด้วยว่าหน้านี้ใช้ API เส้นไหนมาทำงาน ระบบทำงานยังไง ดึงข้อมูลยังไงมีผลดียังไงบ้าง
  ## การออกแบบ Database และ API จะต้องเก็บข้อมูลใน Documentation ด้วยเพื่อตรวจเช็คว่าระบบที่กำลังจะพัฒนามีการสร้าง API ไว้แล้วหรือยัง หรือสร้าง database ไว้แล้วหรือยังเพื่อป้องกันไม่ให้ระบบ Cursor AI Agent Mode สร้างระบบเดิมซ้ำเข้าไปทำให้ระบบหลังบ้านทำงานหนักและผิดพลาดได้ในอนาคต
